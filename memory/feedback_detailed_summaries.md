> EXAMPLE memory of type `feedback`. Mirrors [[RULE-001]] in the knowledge map.

After any significant work, deliver a comprehensive summary: which files changed and why, how the pieces
fit, how to run it, what to know going forward, and the caveats.

**Why:** the owner needs full visibility without a round-trip of follow-up questions.

**How to apply:** end substantive turns with the summary; skip it only for trivial one-liners.

> EXAMPLE project topic. Each real project gets a folder under `knowledge/projects/<slug>/` with a
> `PROJ-NNN_overview.md`, a chronological `progress.md`, and per-feature `PROJ-NNN_<feature>.md` files.

## What it is
One-paragraph description of the project, its purpose, and who it's for.

## Stack
- Language / framework / runtime
- Hosting / deploy target
- Key external services

## Where it lives
- Repo path / URL
- Live URL(s)
- Service names (systemd unit, container, etc.)

## Hard constraints
- Non-obvious invariants the assistant must never violate (data shape, auth model, branding rules, …).

## Status
What's live, what's frozen, what's the next open thread. Keep the running log in `progress.md`.
